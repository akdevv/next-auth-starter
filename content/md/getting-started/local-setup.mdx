# üì¶ Local Setup & Installation

This guide will help you get the next-auth-starter project up and running on your local machine using Next.js, TypeScript, Bun, Prisma, and Supabase.

## üìã Prerequisites

Before getting started, make sure you have:

- **Node.js 18+** or **Bun** installed
- **Git** for cloning the repository
- **Supabase account** for database hosting
- **Google Cloud Console** account (for OAuth)
- **Resend account** for email services

## 3Ô∏è‚É£ Set Up Environment Variables

Create a `.env` file in the root directory:

```bash
cp .env.example .env
```

Fill in your environment variables (see [Environment Variables](/docs/getting-started/environment-variables) for details):

```env
NEXT_PUBLIC_BASE_URL="http://localhost:3000"
AUTH_SECRET="your-secret-key"
GOOGLE_CLIENT_ID="your-google-client-id"
GOOGLE_CLIENT_SECRET="your-google-client-secret"
NEXT_PUBLIC_RESEND_API_KEY="your-resend-api-key"
DATABASE_URL="your-supabase-db-url"
```

```bash

# Push schema to database

# (Optional) Run migrations for production
bun prisma migrate dev --name init
```

## üö® Troubleshooting

**Database connection issues:**
- Verify your `DATABASE_URL` in `.env`
- Check Supabase project is active
- Run `bun prisma db push` again

**Build errors:**
- Clear `.next` folder: `rm -rf .next`
- Reinstall dependencies: `rm -rf node_modules && bun install`

**Port already in use:**
- Change port: `bun dev --port 3001`
- Or kill existing process: `lsof -ti:3000 | xargs kill`
